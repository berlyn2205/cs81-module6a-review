# Reflection

## What was the hardest part to understand?

The hardest part was understanding how prototype methods work and how they connect to the Playlist object. It took time to understand that methods can be added to an object's prototype instead of being written directly inside the constructor.

## How does this code use this to tie data and behavior together?

The keyword this refers to the current Playlist object. It allows each method to access and modify the playlist's data, such as the name, songs, and currentSong properties.

## What would you do differently if you wrote this object from scratch?

If I created this object from scratch, I would add more features like removing songs, searching for songs, and showing the current song. I would also organize the code using modern JavaScript classes.
